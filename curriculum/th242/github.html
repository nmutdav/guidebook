Follow these steps to upload your `.pde` sketches and include documentation in your GitHub repository. Proper documentation helps keep track of your work and provides valuable context for your projects.

Important Note on GitHub

GitHub is a public platform, so while only team members can commit to their respective repositories, please:

  • **Avoid sharing any personal or sensitive information** in your commits or files.
  • **Maintain proper and school-appropriate etiquette** when using the platform. Always be respectful and professional in your interactions and content.

1. Open GitHub Desktop

  • Launch GitHub Desktop.
  • Ensure you are signed in with your GitHub account.

2. Clone Your Repository (If Not Already Done)

  • In GitHub Desktop, go to File > Clone repository.
  • Select the repository you have been provided (e.g., "Group 1" or "Group 2").
  • Choose a local path on your computer where the repository will be saved.
  • Click Clone to download the repository to your computer.

3. Add Your `.pde` Sketches

  • Open the folder where you cloned your repository on your computer.
  • Copy your `.pde` sketches into this folder.

4. Create Documentation Files

  • In the same repository folder, create a new file named README.md for Markdown or README.txt for plain text.
  • Include your project documentation in this file. This can include:
    • Your reflections on the project
    • Proposals or plans for future work
    • Instructions for running or using the sketches
  • For Markdown files, you can use simple formatting like:
    • # Project Title
## Reflections
Describe your reflections here.

## Proposals
Outline your proposals or future plans here.

## Instructions
Provide instructions on how to use or run the sketches here.

5. Commit Your Changes

  • Return to GitHub Desktop. You should see your `.pde` files and `README` file listed under the Changes tab.
  • Write a brief commit message describing your changes (e.g., "Added initial sketches and documentation").
  • Click Commit to main to save your changes locally.

6. Push Your Changes to GitHub

  • After committing your changes, click on Push origin in GitHub Desktop. This uploads your local commits to the GitHub repository.

7. Verify Your Upload

  • Go to your repository on GitHub's website.
  • Ensure your `.pde` sketches and `README` file are visible in the repository files.

8. Additional Tips

  • Keep your documentation clear and concise to provide valuable context for your work.
  • Only make changes to the files you need to work on. Avoid creating or switching branches to keep things simple.
  • If you encounter any issues, ask for help or refer to the GitHub documentation.

By following these steps, you'll be able to upload and manage your sketches and documentation effectively. If you have any questions, feel free to reach out!
