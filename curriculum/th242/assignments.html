Assignments

In this class, we will use the Processing IDE to explore multimedia projects integrating audio and visual elements. Each assignment involves team-based collaboration with rotating members to help you build and refine your skills in creative coding. Individual contributions are required for each assignment to ensure personal accountability. We will use GitHub for collaboration, and each team will be provided a repository to build upon to host and showcase their projects.

Team Grouping and Roles

  • Team Formation: Teams will be formed for each assignment and reorganized periodically to ensure a mix of experiences and perspectives.
  • Roles:
    • Project Lead: Organizes team meetings, manages project assets, and oversees project progress.
    • Resource Curator: Gathers and organizes images, audio, and other assets relevant to the project.
    • Code Developer: Focuses on implementing the coding aspects of the project, including writing and debugging code.
    • Documenter: Maintains project documentation, including README files and assignment reflections.

Presentations

Presentations for each assignment should last between 10 to 15 minutes, followed by additional time for Q&A. Ensure your presentation effectively communicates your project's objectives, implementation, and outcomes.

Grading Breakdown

Each assignment will be evaluated based on the following criteria:

  • Project Implementation: 25%
  • Creativity and Innovation: 25%
  • User Interaction: 25%
  • Documentation and Quality: 25%

The final grade for each assignment will be based on a combination of these factors. Detailed feedback will be provided to help you improve and succeed in future assignments.

Assignments

Assignment 1: Audio-Visual Mood Board

  • Objective: Create a simple visual representation that reacts to an audio file, conveying a specific mood or emotion.
  • Tasks:
    • Choose a short audio clip that evokes a clear mood (e.g., calming, intense, joyful).
    • Create a visual animation that responds to the audio, such as changing colors or shapes based on the audio's amplitude.
    • Ensure the visuals match the mood of the audio clip, e.g., calming visuals for a soothing sound.
  • Reflection: Submit a brief text file reflecting on how the audio influenced your visual design and what challenges you encountered.
  • GitHub Submission: Ensure your project is hosted on GitHub with clear instructions on how to run your code.

Assignment 2: Interactive Soundscape

  • Objective: Develop an interactive visual environment where users can manipulate sounds and visuals in real-time.
  • Tasks:
    • Create a visual environment where different parts of the screen produce different sounds when interacted with.
    • Allow users to modify the visuals by interacting with the audio elements, such as changing colors or patterns.
    • Make sure the interaction is intuitive and engaging.
  • Reflection: Write a reflection on how users interact with your project and how this interaction affects their experience.
  • GitHub Submission: Host your project on GitHub and include a README file with instructions for running the code.

Assignment 3: Dynamic Visual Symphony

  • Objective: Create a visual system that translates various audio properties (e.g., pitch, tempo) into complex visual patterns and effects.
  • Tasks:
    • Use an audio file with varied elements (e.g., music with different instruments and rhythms).
    • Develop visual effects that change dynamically in response to different audio properties like pitch and tempo.
    • Create a visual pattern that evolves as the audio plays, reflecting the audio's complexity.
  • Reflection: Discuss how the audio properties influenced your visual design and any difficulties you faced in creating dynamic effects.
  • GitHub Submission: Ensure your project is properly documented on GitHub, including setup and running instructions.

Final Assignment: Immersive Audio-Visual Experience

  • Objective: Collaborate on a comprehensive project that combines all techniques learned to create an immersive experience integrating complex audio-visual elements.
  • Tasks:
    • Design an immersive environment that uses advanced audio analysis and visual synchronization.
    • Include user interactions that allow for a personalized experience, such as controlling the environment through audio input.
    • Ensure the final project is polished, cohesive, and provides a captivating user experience.
  • Reflection: Submit a detailed reflection on the overall project development process, including team dynamics, challenges faced, and how well the project achieved its immersive goals.
  • GitHub Submission: Host your final project on GitHub with comprehensive documentation, including a user guide and any necessary instructions.
