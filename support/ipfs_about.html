What is IPFS?

IPFS is a peer-to-peer hypermedia protocol designed to make the web faster, safer, and more open. Instead of relying on a centralized server, IPFS uses a distributed network to store and retrieve files. Each file is identified by its content, making it easy to share and retrieve. Here’s how it works:

  1. Upload the File: You select a file on your computer and add it to IPFS.
  2. Chunking: The file is broken into smaller pieces, called blocks.
  3. Hashing: Each block gets a unique ID (CID) based on its content.
  4. Storage: The blocks are stored on your local IPFS node.
  5. Distributed Network: The blocks are shared with other nodes in the IPFS network.
  6. Content Addressing: You can access the file using its CID, no matter where it's stored.
  7. Persistence: As long as at least one node has the file, it remains available on IPFS.

If you have multiple nodes (like two computers), and one goes offline, you can still retrieve your files from the other node, as long as they were shared between them.

Installing IPFS on Your Desktop

  1. Visit the IPFS Installation Page.
  2. Choose the appropriate version for your operating system (Windows, macOS, or Linux).
  3. Follow the installation instructions provided on the page.
  4. After installation, open your terminal or command prompt.
  5. Run the following command to initialize IPFS:
    6. ipfs init
  6. Start the IPFS daemon by running:
    7. ipfs daemon

Adding Files to IPFS

To add files to IPFS, follow these steps:

  1. Open your terminal or command prompt.
  2. Navigate to the directory containing the files you want to back up.
  3. Add a file to IPFS using the command:
    4. ipfs add yourfile.ext
  4. IPFS will return a unique hash (CID) for your file. This hash is how your file is identified on the IPFS network.

Retrieving Files from IPFS

To retrieve your files from IPFS, use the following command:

ipfs cat yourfileCID

You can also view the file in a web browser by visiting:

https://ipfs.io/ipfs/yourfileCID

Benefits of More Users in IPFS

Encouraging more users within a university to adopt IPFS offers several advantages:

  1. Increased Redundancy: More users mean that files are replicated across multiple nodes, enhancing availability and resilience.
  2. Improved Accessibility: A larger network results in faster retrieval times as files can be accessed from various locations.
  3. Decentralized Storage: Spreading storage across many users reduces reliance on central servers, minimizing the risk of data loss.
  4. Collaborative Projects: Students can easily share files with peers, streamlining group work through unique IPFS hashes.
  5. Reduced Bandwidth Costs: Distributed hosting decreases the burden on any single user’s bandwidth, improving performance.
  6. Incentivizing Participation: Users may benefit from faster access to shared resources, encouraging more participation.
  7. Enhanced Security: Files are identified by content hashes, reducing the risk of unauthorized changes.
  8. Facilitated Long-term Access: Multiple users storing the same content ensures long-term availability of important files.

Best Practices

  • Regularly back up your important files to IPFS.
  • Share the CID with peers for collaborative projects.
  • Consider using IPFS in conjunction with other backup solutions.
  • Keep your IPFS node running to ensure your files are accessible.

Conclusion

Using IPFS as a backup drive can provide a reliable and decentralized way to store your university projects. With just a few commands, you can ensure your important files are safe and accessible anytime, anywhere.
